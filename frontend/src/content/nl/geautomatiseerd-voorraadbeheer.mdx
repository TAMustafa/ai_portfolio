## Uitdaging
De klant had meerdere verkoopkanalen (webshop, marketplace, fysieke winkels) zonder eenduidige voorraadbron. Dit leidde tot:

- Dubbele bestellingen en uitverkochte artikelen
- Tijdrovende handmatige afstemming
- Verlies van omzet door onnauwkeurige voorraadstanden

<Callout type="warning" title="Belangrijk knelpunt">
Parallelle verkopen op verschillende kanalen veroorzaakten inconsistenties binnen enkele minuten.
</Callout>

## Oplossing
We implementeerden een event-gedreven synchronisatie-laag bovenop bestaande systemen. Elke verkoop, retour of correctie publiceert een event dat asynchroon verwerkt wordt door een lightweight worker. De worker normaliseert gegevens en werkt één bron van waarheid bij.

```ts
// Voorbeeld: order-consumer (vereenvoudigd)
type OrderEvent = { id: string; sku: string; qty: number; type: 'SALE' | 'RETURN' };

export async function consume(event: OrderEvent) {
  const stock = await inventoryRepo.getBySku(event.sku);
  const newQty = event.type === 'SALE' ? stock.qty - event.qty : stock.qty + event.qty;
  await inventoryRepo.setQty(event.sku, newQty);
  await publish('inventory.updated', { sku: event.sku, qty: newQty });
}
```

<Callout type="info" title="Architectuur">
Event-gedreven integraties met retries, idempotente handlers en observability (metrics + structured logging).
</Callout>

## Resultaat
- 40% minder out-of-stock
- Minder handwerk; updates binnen seconden zichtbaar op alle kanalen
- Betrouwbare rapportage dankzij één bron van waarheid

<Callout type="success" title="Impact op KPI’s">
Hogere leverbetrouwbaarheid en betere klantervaring door accurate voorraad.
</Callout>
